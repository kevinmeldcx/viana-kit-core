Strategic Phased Rollout

To avoid executing everything at once, we propose a three-phase development cycle. Phase 1 establishes the strict JSON Schema required for the advanced tooling in Phases 2 and 3.

Research Plan & Suggestions

Based on your role as Creative Director, bridging design and development via AI is the primary differentiator of Viana Kit. Here are specific suggestions for your plan.

Suggestion 1: The MCP Strategy (AI UI Gen)

To allow Gemini/Claude to use Viana Kit, you must develop an MCP (Model Context Protocol) server. Rather than just giving the AI raw code, your MCP should expose a structured tool like `generate_viana_ui(component_list)`. The MCP server responds with your exact shadcn customizations and prop constraints so the AI never hallucinates incorrect Tailwind classes.

Suggestion 2: paper.design Integration

paper.design acts as a React-to-Design bridge. You should maintain a single source of truth for component `props`. Create a metadata JSON file for every shadcn component. This file informs both the MCP (what the AI can use) and paper.design (what variations the designer can drag-and-drop).

Suggestion 3: Monorepo Structure

Use Turborepo or Nx. Structure your repository with packages for: `@viana/ui` (React components), `@viana/tokens` (CSS variables/Tailwind config), and apps for `@viana/docs` and `@viana/mcp-server`. This ensures all outputs update simultaneously when branding changes.
